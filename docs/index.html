The first systematic JavaScript vulnerability benchmark

SecJS bundles ArenaJS (a large-scale JS vulnerability dataset) with ForgeJS and JudgeJS, two automation frameworks that generate realistic projects, evaluate LLMs, and report reproducible metrics.

Project Overview

The benchmark targets research teams that need curated JavaScript projects, function-level ground truth, and standardized LLM evaluation protocols. ArenaJS delivers 1,852 real projects, ForgeJS automates data generation, and JudgeJS runs consistent vulnerability assessments across five dataset variants.

Variant Summary

Variant Projects Description
Original 1,852 Clean OSS projects without augmentation.
Noise ~1,834 Injects spurious code/comments to evaluate resilience to clutter.

Getting Started

Install

# Clone & install git clone https://github.com/SecJS-Vuln-Benchmark/SecJS-Benchmark.git cd SecJS-Benchmark pip install -r requirements.txt

JudgeJS Evaluation

# Run an evaluation python main.py --model gpt-4 --dataset original # Summaries and charts python analyze_stats.py

ForgeJS Generation

# Generate a single variant python generate_dataset.py --strategy noise # Generate every variant python generate_dataset.py --strategy all

Dataset Availability

The dataset download links will be published in the repository releases. Each release will include:

  • Original and augmented project archives
  • Function-level ground truth files
  • Metadata (project description, CWE labels, provenance)
  • Evaluation scripts for JudgeJS
